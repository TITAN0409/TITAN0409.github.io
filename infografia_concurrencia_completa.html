Programación Concurrente:
El Arte de la Multitarea

La capacidad de un programa para gestionar múltiples tareas de forma intercalada. No es necesariamente que ocurran al mismo instante exacto — eso es paralelismo — sino que el sistema progresa en varias tareas simultáneamente.

🧵
¿Qué es un Hilo (Thread)?
Un hilo es la unidad más pequeña de ejecución que un sistema operativo puede gestionar de forma independiente.
🏢

El Proceso

Es el "contenedor" — como una oficina entera con todos sus recursos: memoria, archivos abiertos y conexiones de red.

👷

El Hilo

Es el "trabajador" dentro de esa oficina. Un proceso puede tener múltiples hilos ejecutando tareas diferentes al mismo tiempo.

🔗

Memoria compartida

Todos los hilos de un mismo proceso comparten la misma memoria — permite comunicación rápida pero requiere sincronización cuidadosa.

¿Cómo funciona?
1

Context Switching: El procesador cambia tan rápido entre hilos que parece que todo sucede a la vez. Cada cambio guarda el estado del hilo actual y restaura el del siguiente.

2

Eficiencia de I/O: Mientras un hilo espera una respuesta de internet o disco, otro hilo sigue haciendo cálculos — el CPU nunca queda inactivo.

3

Scheduler del SO: El sistema operativo decide qué hilo corre en cada instante usando algoritmos de planificación como Round Robin o prioridades.

Ventajas vs Desafíos

✓ Ventajas

Mayor responsividad: La interfaz no se congela mientras se procesan datos en segundo plano.

Aprovechamiento de CPU: Uso óptimo de procesadores multinúcleo modernos.

División de tareas: Estructura de código más lógica para tareas independientes.

Menor costo: Crear hilos es más barato que crear procesos nuevos.

✗ Desafíos

Race Conditions: Dos hilos modifican el mismo dato al mismo tiempo con resultados impredecibles.

Deadlocks: Hilos que se esperan mutuamente y quedan bloqueados para siempre.

Depuración difícil: Los errores son no determinísticos y difíciles de reproducir.

Starvation: Un hilo con baja prioridad nunca llega a ejecutarse.

🛡️
Herramientas de sincronización
Para evitar el caos, los programadores usamos mecanismos de control que regulan el acceso a recursos compartidos:
SOLUCIÓN

Mutex (Exclusión Mutua)

Un "candado" que solo permite que un hilo acceda a un recurso a la vez. El resto espera en cola.

SOLUCIÓN

Semáforos

Controlan el acceso basado en un contador de permisos. Permiten N hilos simultáneos, no solo 1.

SOLUCIÓN

Monitores

Estructuras de alto nivel que encapsulan variables y métodos permitiendo acceso seguro automático.

PROBLEMA

Deadlock — Cómo evitarlo

Definir un orden fijo para adquirir locks, usar timeouts, y nunca mantener un lock al pedir otro.

Otros usos en el mundo real
Videojuegos

Hilo de física, hilo de audio, hilo de render y hilo de red corriendo simultáneamente.

Servidores web

Cada solicitud HTTP recibe un hilo propio para responder a miles de usuarios a la vez.

Apps móviles

El hilo principal gestiona la UI; hilos de fondo descargan datos sin congelar la pantalla.

💻
Ejemplo en código — Python
import threading
import time

def descargar_imagen(nombre):
  print(f"Descargando {nombre}...")
  time.sleep(2) # simula descarga
  print(f"{nombre} lista")

# Crear 3 hilos que corren en paralelo
for img in ["foto1", "foto2", "foto3"]:
  t = threading.Thread(target=descargar_imagen, args=(img,))
  t.start() # lanzar hilo

# Con hilos: ~2s. Sin hilos: ~6s.
